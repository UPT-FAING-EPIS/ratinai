RF-01 — Crear cuenta médico

|CASO DE USO —Crear cuenta médico  |  CU-01||
| :- | :- |
|Actor|Administrador|
|Descripción|Permite al Administrador del establecimiento registrar la cuenta de un médico oftalmólogo dentro de la plataforma RetinAI. A través de un formulario dedicado, el Administrador ingresa los datos profesionales del médico (nombre completo, correo electrónico, número CMP, especialidad y contraseña temporal), tras lo cual el sistema crea la cuenta de manera inmediata con estado «Activo» y envía un correo electrónico de bienvenida al médico con sus credenciales de primer acceso.|
|Precondiciones|<p>El Administrador tiene sesión activa en la plataforma RetinAI.</p><p>El Administrador dispone del nombre completo, correo electrónico, número CMP y especialidad del médico a registrar.</p><p>El correo electrónico y el número CMP del médico no están registrados previamente en el sistema.</p><p>El sistema se encuentra operativo con conexión a base de datos y servicio de correo disponible.</p>|
|Postcondiciones|<p>La cuenta del médico queda creada con estado «Activo» en la base de datos del establecimiento.</p><p>El médico recibe un correo electrónico con su contraseña temporal de primer acceso.</p><p>La cuenta queda marcada como «Clave temporal», indicando que el médico deberá cambiarla en su primer ingreso.</p><p>El médico aparece en la tabla «Médicos activos» del panel del Administrador.</p>|
|FLUJO NORMAL||
|Acción del actor|Curso del sistema|
|1\. El Administrador, con sesión activa, selecciona la opción «Médicos» en el menú lateral izquierdo bajo la sección «GESTIÓN».|2\. El sistema carga la vista «Médicos activos» mostrando la tabla de médicos registrados en el establecimiento (Hospital de la Solidaridad). La tabla incluye las columnas: Médico (nombre + correo), CMP, Especialidad, Último Acceso, Estado y Acción. En la esquina superior derecha aparece el botón azul «Agregar Médico».|
|3\. El Administrador hace clic en el botón «Agregar Médico» ubicado en la parte superior derecha del panel.|4\. El sistema navega a la vista de registro. Aparece una tarjeta blanca centrada con el encabezado «Registrar nuevo médico» acompañado del subítulo «El médico recibirá una contraseña temporal para su primer acceso.» Sobre el formulario se muestra un banner informativo azul: «El médico se creará como Activo con contraseña temporal. Deberá cambiarla en su primer ingreso.»|
|5\. El Administrador completa los siguientes campos del formulario:  • Nombre completo: «Dr. Kalid» • Correo electrónico: correo válido y no registrado • Número CMP: número de colegiatura (ej. 125674) • Especialidad: «Glaucoma» • Contraseña temporal: mínimo 6 caracteres (ej. kalid123)|6\. El sistema refleja en tiempo real el texto ingresado en cada campo. El campo «Contraseña temporal» oculta los caracteres por defecto y los muestra al activar el ícono de ojo. Bajo el campo CMP aparece la leyenda «Colegio Médico del Perú» como referencia contextual. El campo de contraseña muestra el hint «Mínimo 6 caracteres».|
|7\. El Administrador presiona el botón «Guardar médico».|8\. El sistema valida que todos los campos obligatorios estén completos y que el correo electrónico tenga formato válido (ej. usuario@dominio.com). El campo de contraseña debe tener al menos 6 caracteres. Si todo es correcto, el sistema avanza al Paso 9. Si hay errores, ejecuta el Flujo Alternativo A.|
||9\. El sistema consulta la base de datos del establecimiento para verificar que el número CMP y el correo electrónico ingresados no existan previamente. Si alguno ya está registrado, ejecuta el Flujo Alternativo B. Si ambos son únicos,sigue.|
||10\. El sistema crea la cuenta del médico con estado «Activo» y la marca como «Clave temporal». Genera el registro en la base de datos del establecimiento y, de forma simultánea, envía un correo electrónico al correo registrado del médico. El correo tiene asunto «Acceso a RetinAI - Su cuenta ha sido creada» e incluye: banner de marca RetinAI en azul, saludo personalizado «¡Bienvenido, Dr. [Nombre]!», descripción del rol asignado «Médico Oftalmólogo», la contraseña temporal en formato destacado (tipografía monoespaciada), botón «Iniciar Sesión Ahora» y aviso en rojo: «Importante: Por razones de seguridad, el sistema le solicitará cambiar esta contraseña temporal por una nueva en su primer acceso.»|
||11\. El sistema muestra el mensaje de confirmación: «Cuenta creada exitosamente. Se ha enviado una contraseña temporal al correo del médico.» La vista regresa al panel «Médicos activos», donde el nuevo médico aparece en la tabla con estado «Activo» y la etiqueta «Clave temporal» en color naranja, indicando que aún no ha cambiado su contraseña.|
|FLUJO ALTERNATIVO — Datos Incompletos o Formato Inválido||
|Acción del actor|Curso del sistema|
|3A. El Administrador deja uno o más campos vacíos o ingresa un correo con formato inválido (ej. «kalid@» o «kalid.com»).|4A. El sistema detecta los campos con error al hacer clic en «Guardar médico». Muestra indicadores visuales de error directamente en cada campo incorrecto (borde rojo, mensaje bajo el campo). El campo de contraseña muestra el hint «Mínimo 6 caracteres» si no llega al mínimo requerido. El registro no se crea ni se realiza ninguna consulta a la base de datos.|
|5A. El Administrador corrige los campos indicados.|El sistema elimina los indicadores de error conforme el usuario corrige cada campo y los valida en tiempo real. Una vez todos los campos son válidos, el Administrador puede volver a hacer clic en «Guardar médico» para continuar con el Paso 4 del flujo normal.|
|FLUJO ALTERNATIVO — CMP o Correo ya Registrado||
|Acción del actor|Curso del sistema|
|3B. El Administrador ingresa un número CMP o correo electrónico que ya está registrado en el sistema (ej. el correo gg2022074263@virtual.upt.pe ya existe).|5B. El sistema muestra un banner de error en la parte superior del formulario con fondo rojo claro: «⚠ Ya existe un usuario con ese correo electrónico.» (o mensaje equivalente para CMP duplicado). Todos los campos previamente ingresados se conservan. El registro no se crea. El Administrador puede corregir el dato duplicado y reenviar el formulario.|
|5B.1 El Administrador corrige el correo o CMP por un valor único.|El sistema elimina el banner de error y permite que el Administrador vuelva a intentar el registro desde el Paso 4 del flujo normal.|
|EXCEPCIONES||
|Error al enviar correo: Si el sistema no puede enviar la contraseña temporal al correo del médico (por fallo en el servicio SMTP, tiempo de espera agotado u otro error de conectividad), la cuenta se crea igualmente con estado «Activo» y marca «Clave temporal». El sistema muestra al Administrador la contraseña temporal directamente en pantalla, dentro de un cuadro destacado, para que pueda entregársela al médico de forma manual. Se registra internamente el fallo de envío de correo para auditoría.||
|Error al guardar: Si el sistema no puede completar la escritura en la base de datos por un error técnico (fallo de conexión, timeout, error de servidor), muestra el mensaje: «No se pudo crear la cuenta. Intente nuevamente.» No se crea ningún registro parcial ni se envía ningún correo. Todos los datos ingresados en el formulario se conservan para que el Administrador pueda reintentar la operación sin tener que completar el formulario nuevamente.||
